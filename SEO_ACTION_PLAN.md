# План SEO-оптимизации проекта (Next.js + Django)

Стратегия создания мощного SEO-ядра с фокусом на ключевые запросы: **"Разработка сайтов"**, **"Администрирование и обслуживание корпоративных IT инфраструктуры"**, **"Интеграция ИИ агентов и автоматизация бизнес процессов"**.

## Ответы на ваши вопросы:

> **Как мои статьи в блоге будут учавствовать в SEO?**
Статьи — это ваш главный "магнит" для трафика. Каждая статья будет привлекать людей (потенциальных клиентов) по узким запросам (например: "Как настроить сервер", "Внедрение ИИ в отдел продаж"). Человек ищет решение проблемы, находит вашу статью, читает экспертный материал и затем переходит к вашим услугам.

> **Стоит ли их делать как отдельные страницы сайта?**
**Да, 100% обязательно!** Поисковики (Google, Yandex) ранжируют не "сайт целиком", а конкретные страницы. Я проверил вашу архитектуру — у вас уже есть роут `/blog/[slug]/page.tsx`, что отлично! Нам нужно лишь добавить туда **динамическую генерацию метаданных** (чтобы заголовок из базы данных автоматически становился `<title>` страницы).

---

## Предлагаемый План Действий

### 1. Внедрение Глобального SEO-Ядра (layout.tsx)
Мы удалим стандартные настройки `Create Next App` в главном layout и заменим их на мощные метавеличины (название, описание, open_graph для соцсетей), которые будут содержать ваши главные ключевые слова.

**Файл:** `frontend/src/app/layout.tsx`
- Интеграция `metadata` с `title.template` (чтобы на всех страницах было `Название Страницы | Ваше Имя`).
- Внедрение тегов: Разработка сайтов, IT-обслуживание корпоративной инфраструктуры, Интеграция ИИ.
- Подключение мета-тегов Open Graph для красивых превью в Telegram/WhatsApp.

### 2. Динамическое SEO для Блога
Поисковики должны видеть уникальный заголовок и уникальное описание для каждой вашей статьи.

**Файл:** `frontend/src/app/blog/[slug]/page.tsx`
- Внедрение функции `generateMetadata()` (фишка Next.js App Router), которая будет запрашивать статью у вашего Django REST API и на лету устанавливать параметры `<title>`, `<meta name="description">` и `<meta name="keywords">`.

### 3. SEO для статических страниц
Каждой существующей странице мы зададим уникальный фокус (например, страница проектов будет ранжироваться под "разработка сайтов", а индексная страница под "интеграция ИИ").

**Файлы:** `frontend/src/app/page.tsx` (Главная) и `frontend/src/app/blog/page.tsx` (Список статей)
- Внедрение кастомного `metadata` на каждую страницу.

### 4. Генерация Sitemap и Robots.txt
Без этих файлов поисковые роботы Яндекса и Google не будут понимать, как правильно сканировать ваш сайт, и они могут проигнорировать статьи.

**Новые файлы:**
- `frontend/src/app/sitemap.ts`: Динамическая генерация файла `sitemap.xml`, который будет брать все слаговые (slug) ссылки ваших статей из Django базы данных + статические страницы.
- `frontend/src/app/robots.ts`: Настройка файла `robots.txt`.
